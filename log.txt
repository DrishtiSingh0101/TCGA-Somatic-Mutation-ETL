Step 2: Extract TCGA Somatic Mutation Data

Now let's build the first ETL component.

Understanding TCGA

TCGA data is hosted by:

National Cancer Institute's GDC Portal.

We will use:

https://api.gdc.cancer.gov

The API is free.

No login required.

What Data Are We Extracting?

For each cancer type:

1. BRCA

2. LUAD

3. COAD

we want:

| Column                 | Meaning                |
| ---------------------- | ---------------------- |
| case_id                | Patient ID             |
| gene                   | Gene Name              |
| chromosome             | Chromosome             |
| start_position         | Variant Position       |
| variant_classification | Missense, Nonsense etc |
| reference_allele       | REF                    |
| tumor_allele           | ALT                    |

Required Packages

Install:

pip install requests pandas tqdm

create requirements.txt

First Extraction Script

Create:

# src/extract.py

What This Script Does

It DOES NOT download mutation data yet.

It first teaches an important ETL concept:

Extract Metadata

We ask GDC:
Show me all MAF files
for TCGA-BRCA

GDC returns:

{
  "file_id": "...",
  "file_name": "...",
  "file_size": ...
}

This is the "inventory" step.

Most ETL pipelines do this first.

Varify extraction


Why This Is Better Than Downloading First

Most beginners:

Download huge files
↓
Figure out what they are

Real ETL:

Discover metadata
↓
Validate metadata
↓
Download selected files
↓
Transform
↓
Load
